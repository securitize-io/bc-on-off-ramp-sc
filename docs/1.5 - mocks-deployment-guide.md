# Mock Contracts Deployment Guide

This guide provides a streamlined approach to deploy mock contracts and create test wallets for the Securitize Off-Ramp system.

## Overview

This workflow allows you to:

- Deploy mock contracts for testing
- Create test wallets with ETH funding
- Mint test tokens for your wallets

## Prerequisites

Ensure you have completed the [Requirements](1%20-%20Requirements.md) guide first to set up your `.env` file.

## Quick Start Workflow

### 1. Create Test Wallets

Create wallets for testing (automatically saves to `.env.wallets`):

```bash
# Create provider wallet with 0.05 ETH (funds liquidity pool)
npx hardhat --network sepolia create-wallet --name PROVIDER --fund 0.05

# Create recipient wallet with 0.05 ETH (receives liquidity tokens)
npx hardhat --network sepolia create-wallet --name RECIPIENT

# Create fee collector wallet with 0.05 ETH (receives fees)
npx hardhat --network sepolia create-wallet --name FEE_COLLECTOR

# Create investor wallet with 0.05 ETH (holds DS tokens and performs redemptions)
npx hardhat --network sepolia create-wallet --name INVESTOR --fund 0.05
```

**Generated wallets are saved to `.env.wallets` with the format `{NAME}_ADDRESS` and `{NAME}_PRIVATE_KEY`.**

### 2. Deploy Mock Contracts

Deploy the required mock contracts for testing. Each command will output a contract address that you'll need for the next steps:

```bash
# Deploy NAV Provider with a default rate (1 DS token = 1 USDC)
npx hardhat --network sepolia deploy-contract --contract-name MockSecuritizeInternalNavProvider 10000

# Deploy Fee Manager - replace FEE_COLLECTOR_ADDRESS with the address from step 1
npx hardhat --network sepolia deploy-contract --contract-name MockFeeManager $FEE_COLLECTOR_ADDRESS

# Deploy Trust and Registry services (no parameters needed)
npx hardhat --network sepolia deploy-contract --contract-name MockTrustService
npx hardhat --network sepolia deploy-contract --contract-name MockRegistryService

# Deploy USDC mock token (this will be your liquidity token)
npx hardhat --network sepolia deploy-contract --contract-name MockERC20 "Mock USDC" "MUSDC" 6

# Deploy DS Token - replace TRUST_SERVICE_ADDRESS with the address from Trust Service deployment
npx hardhat --network sepolia deploy-contract --contract-name MockDSToken "Mock DS Token" "MDST" 6 $REGISTRY_SERVICE_ADDRESS $TRUST_SERVICE_ADDRESS
```

**Important**: After each deployment, save the contract address from the output. You'll need these addresses for the Off-Ramp deployment.

### 3. Mint Test Tokens

After deployment, you need to mint tokens to your wallets for testing. First, check the addresses created in step 1:

```bash
# Check your wallet addresses
cat .env.wallets
```

You should see output like:

```
PROVIDER_ADDRESS=0x1234...
PROVIDER_PRIVATE_KEY=0xabcd...
INVESTOR_ADDRESS=0x5678...
INVESTOR_PRIVATE_KEY=0xefgh...
...
```

Now mint tokens using the actual addresses (replace the examples below):

```bash
# Mint 1M USDC to provider wallet (so it can provide liquidity)
npx hardhat --network sepolia contract-call \
    --contract-name MockERC20 \
    --contract-address 0xYOUR_USDC_CONTRACT_ADDRESS \
    --method mint 0xYOUR_PROVIDER_ADDRESS "1000000000000"

# Mint 1K DS tokens to investor wallet (so they can redeem them)
npx hardhat --network sepolia contract-call \
    --contract-name MockDSToken \
    --contract-address 0xYOUR_DS_TOKEN_ADDRESS \
    --method mint 0xYOUR_INVESTOR_ADDRESS "1000000000"
```

**Note**: Replace the contract addresses with the actual addresses from step 2, and wallet addresses with those from `.env.wallets`.

### 4. Register Investor and Wallets

After minting tokens, you need to register the investor and wallets in the MockRegistryService for the Off-Ramp to work properly:

```bash
# Register the investor in the MockRegistryService
npx hardhat --network sepolia register-investor \
    --registry-address 0xYOUR_REGISTRY_SERVICE_ADDRESS \
    --investor-address 0xYOUR_INVESTOR_ADDRESS \
    --investor-id "123" --country "AR"
```

## Configure deployment-config.json

After deploying mocks, creating wallets, and registering the investor, update your `deployment-config.json` file with the deployed contract addresses and wallet addresses from `.env.wallets`.

**🤖 AI:** Offers the user to create a `deployment-config.json` following the structure of the example file.
